# 中文场景文字识别实验报告 - 失败分析与改进建议

**日期**: 2026-03-31
**实验状态**: 负优化（28分 -> 22分）

## 1. 实验回顾
本次实验引入了以下三项重大改变：
1. **分阶段数据增强**: 引入了 `light` (亮度/对比度) 和 `heavy` (高斯模糊/椒盐噪声) 模式。
2. **学习率策略**: 引入了 `Warmup` (预热) + `Cosine Annealing` (余弦退火) 以及 `AdamW`。
3. **训练规模**: Batch Size 提升至 256。

## 2. 失败原因深度分析 (根因排查)

### A. 骨干网络 (Backbone) 与数据增强的“供需失衡” (主因)
* **现状**: 目前的 Backbone 仅为一个 `Conv2D(kernel=4, stride=4)`。
* **问题**: 
    * 这种单层卷积就像一个远视眼，本来认字就费劲。
    * 当我们加入“模糊”和“噪声”增强后，图像的局部笔画细节被大量破坏。
    * 由于 Backbone 深度不够（感受野极小），它无法从受损的像素中提取出有意义的特征，传给 Transformer 的特征向量充满了随机噪声。
* **结论**: **模型在 Heavy 阶段不仅没学到抗压，反而被“带偏”了。**

### B. 增强策略与最终收敛的断层
* **问题**: 即使我们在最后 50 Epoch 关闭了增强，但由于之前的 50 轮 Heavy 训练让模型建立了一套错误的特征映射逻辑，短短 50 轮的 Fine-tuning 无法纠正已经偏离轨道太远的权重。

### C. 学习率可能过快地压到了极低值
* **问题**: `CosineAnnealing` 会根据总步数压低 LR。在最后的 Fine-tuning 阶段，如果 LR 已经降到了 `5e-7` 这样极小的量级，模型几乎失去了纠错能力。

## 3. 经验总结：模型改进的优先级错误
我们这次实验先优化了“外功”(数据增强、训练策略)，但由于我们的“内功”(模型架构/Backbone) 实在太弱，导致无法消化高级的增强策略。

**结论**: 在没有强大的 CNN 特征提取器之前，强行上“重度增强”会产生严重的负面效果。

## 4. 下一步逆袭方案 (优先级排序)

1. **[优先级最高] 升级 Backbone**: 
   * 放弃单层卷积，改用 3-4 层带 `BatchNorm` 和 `ReLU` 的小型卷积网络。
   * 这将使模型具备识别噪声中字符残影的能力。
2. **[优先级中] 弱化增强强度**:
   * 将高斯模糊的半径上限从 `1.0` 降至 `0.3`。
   * 甚至在 Backbone 变强之前，先完全停用 `heavy` 模式，只保留 `light` 模式。
3. **[优先级低] 调整 LR Scheduler**:
   * 让微调阶段维持一个略高一点的学习率（如 `1e-5`），确保模型有能力在关闭增强后迅速修正是非。
